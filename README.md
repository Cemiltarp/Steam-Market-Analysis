## 📌 Project Overview
This project is an advanced, automated data pipeline and financial dashboard designed to analyze and forecast the global Steam game market. The project offers a specially developed **Dynamic-Logarithmic ARPU (Average Revenue Per User) Engine**.

By processing real-time Concurrent User (CCU), in-game microtransaction (MTX) tags, and DLC counts, the engine estimates the **Lifetime Total Revenue** and **Current Annual Revenue** of both Live Service giants (e.g., CS2, Dota 2, GTA V) and paid indie games with near-accuracy.

> **Disclaimer:** The financial results presented in this dashboard are estimates generated by a dynamic algorithm based on publicly available API metrics. They do not represent official corporate financial statements.

## The Math Behind the Engine (How it Works)
It calculates current market value using a proprietary algorithm:

**1. Dynamic Active Audience Calculation**
Instead of treating CCU as the total player base, the engine applies a dynamic multiplier based on the scale of the game to estimate daily/monthly active users.

$$Active Players = CCU times CCU Multiplier$$
*(Multiplier varies between 15x and 30x depending on total CCU volume).*

**2. Base ARPU and Logarithmic DLC Contribution**
Base spending habits are determined by the game's business model (Free-to-Play, Premium, MTX availability). DLC contributions are calculated logarithmically to simulate the law of diminishing returns (the more DLC a game has, the less likely a single user is to buy them all).
$$Base ARPU = MTX Base + (8.0 times \ln(1 + Number of DLCs))$$

**3. "Whale Economy" Extreme State Protocol**
Games like *Counter-Strike 2* operate massive loot box and community market economies that traditional ARPU models cannot capture. To correct this, the engine applies a "Whale Multiplier" (up to 2.5x) to games with over 300,000 CCUs, aligning algorithmic predictions with real-world financial reports.
$$Current Annual Revenue = Number of Active Players × (Basic ARPU × Whale Multiplier)$$

## 📌 Key Features

* **Intelligent Data Scraping:** Extracts raw data from the top 5000 games from SteamSpy and Steam APIs, dynamically handling rate caps and anomalies.

* **Automatic Country Detection:** Queries the Twitch (IGDB) API, Wikipedia, and DuckDuckGo to match game developer studios with the countries where their headquarters are located.

* **Live Service and Premium Separation:** Separates past box price sales logic from active live service revenue generation to prevent data pollution.

* **Interactive Streamlit Dashboard:** Visualizes processed data using Plotly; provides dynamic metrics, pie charts for global market shares, and a highly detailed financial data framework.

## Architecture and Pipeline

The data pipeline operates in four consecutive phases:

1. `1_searching.py`: Scans the top 5000 games and creates the underlying dataset.

2. `2_find_country.py`: Enriches the data by identifying each developer's country of origin using a multi-layered API approach.

3. `3_data_structuring.py`: Cleans up anomalies, checks F2P status, MTX availability, and DLC counts directly from the Steam Store API.

4. `4_dashboard.py`: Implements financial formulas and creates the user interface.
